Тема роботи: Програмна система для контролю роботи співробітників служби доставки їжі.


Мета роботи:
На цій лабораторній роботі потрібно розробити мобільний програмний застосунок для будь-якої поширеної на сьогодні мобільної платформи. Мобільний застосунок повинен бути частиною системи і повинен взаємодіяти з нею. Для реалізації програмного застосунку можна використовувати технології для реалізації програмного забезпечення для мобільних платформ, вказані лектором, а саме Kotlin для Android та Swift для iOS.


Хід роботи


1. Діаграма прецедентів (Use Case Diagram) 
Актори: Кур'єр. 
Ключові дії Кур'єра: Авторизація в системі, Перемикання статусу (Онлайн/Офлайн), Перегляд списку замовлень, Перегляд деталей замовлення, Зміна статусу замовлення.
  

Рисунок 1 - UML діаграма прецедентів (Use Case Diagram)
2. Діаграма компонент (Component Diagram) 
Компонент "Mobile App": реалізований на Kotlin (Android). 
Компонент "REST API Server": ядро системи на Node.js (Express).
Компонент "Database": реляційне сховище MariaDB.
  

Рисунок 2 - UML діаграма компонентів (Component Diagram)
3. Діаграма діяльності (Activity Diagram) 
Описує процес завершення замовлення: Натискання кнопки "Завершити" -> Блокування елементів інтерфейсу -> Формування структури запиту -> Відправка запиту на сервер -> Перевірка статусу відповіді -> Відображення результату -> Автоматичне оновлення списку -> Кінець.
  

Рисунок 3 - UML діаграма діяльності (Activity Diagram)
4. Діаграма станів (State Diagram)
Описує життєвий цикл замовлення в системі: Створення (початковий стан) -> Призначення кур'єра (відображення у додатку) -> Успішна доставка (кінцевий стан).
  

Рисунок 4 - UML діаграма станів (State Diagram).




5. Висновки
Під час виконання лабораторної роботи було розроблено мобільний застосунок кур'єра, який успішно інтегровано в загальну клієнт-серверну екосистему проекту. Завдяки використанню бібліотеки Retrofit та Kotlin Coroutines було налаштовано стабільну асинхронну мережеву взаємодію та захищений обмін даними за допомогою JWT-авторизації.
